# Soul of Love Energy Knowledge Base

This folder contains the layered interpretation system for the Daily Energy module.

The goal is to avoid storing hundreds of fixed articles. Instead, readings are generated by combining reusable knowledge layers:

1. Major Arcana card aspect
2. Light or Shadow aspect
3. Aura lens
4. Reading area template
5. Energy score intensity
6. Language mode

## Card Aspect

`cardAspects.js` stores one symbolic knowledge object for each of the 22 Major Arcana cards.

Each card includes:

- core meaning
- light aspect
- shadow aspect
- lesson
- area-specific fragments for love, work, money, warning, and affirmation

Shadow does not mean evil or bad. In Soul of Love language, Shadow means reflective, hidden, unintegrated, tired, or asking for awareness.

## Aura Lens

`auraProfiles.js` stores 12 aura profiles:

- Pearl White
- Solar Gold
- Crimson Red
- Rose Pink
- Emerald Green
- Turquoise
- Sky Blue
- Indigo
- Violet
- Shadow Violet
- Obsidian Black
- Rainbow Prism

Aura is not decorative. Aura changes how the card is interpreted. For example, the same card through Pearl White sounds softer and cleaner, while the same card through Crimson Red emphasizes courage, action, and life force.

## Light / Shadow

The current reading orientation maps into aspects:

- `upright` -> `Light`
- `reversed` -> `Shadow`

Shadow readings should never sound scary or fatalistic. They should help the user notice patterns, boundaries, hidden needs, and areas that want integration.

## Energy Score Intensity

`buildEnergyReading.js` uses score only as an intensity modifier:

- `0-39`: low / quiet / subtle
- `40-69`: medium / balanced / active
- `70-100`: high / strong / radiant

The score does not create separate databases. It changes tone, pace, and emotional state.

## Area Templates

`areaTemplates.js` contains generation templates for:

- universe
- guidance
- love
- work
- money
- focus
- warning
- affirmation
- mission

Each area has its own structure so the reading does not feel like the same paragraph repeated under different labels.

## Language Support

Default output language is `TH`.

`languageMap.js` prepares future modes:

- `TH`
- `EN`
- `MIX`

The UI does not expose a language selector yet. Future modules can route language preference into `buildEnergyReading()`.

## Integration

`src/lib/createReading.js` calls `buildEnergyReading()` and maps the result into the existing reading object fields.

The older layered interpretation engine remains as fallback so the module can evolve safely without breaking the user interface.
